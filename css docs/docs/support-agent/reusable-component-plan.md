# Support Agent Workspace — Reusable Component Plan

This document details the interface contracts, styling variants, and responsive behaviors for all reusable UI components in the Support Agent Workspace.

---

## 1. Conversation Timeline (`ConversationTimeline`)
* **Purpose**: Renders the sequential flow of messages, internal notes, and system log events inside an active conversation.
* **Props**:
  * `messages: Message[]` (array of chat, note, or system message objects)
  * `onScrollToTop: () => void` (callback for pagination/loading history)
  * `isLoadingHistory: boolean`
* **Variants**:
  * Chat: Left/Right aligned message bubbles.
  * Internal Note: Centered full-width purple card block.
  * System Event: Small centered gray text log.
* **Dependencies**: Icons, message bubble rendering blocks
* **Responsive Behavior**: Max-width adapts dynamically; bubbles expand to fit full width on mobile viewports.
* **Theme Behavior**: Agent bubbles use solid blue background in light mode, deep blue in dark mode. System events render in muted gray.
* **RTL Behavior**: Reverses side alignments (Arabic agent replies align left, user replies align right).
* **Accessibility Requirements**: Uses `role="log"`; messages announce chronological additions; images require alternative text.
* **Complexity**: High

---

## 2. AI Suggestion Card (`AISuggestionCard`)
* **Purpose**: Displays a recommended response generated by the local AI helper based on context.
* **Props**:
  * `text: string` (suggested response text)
  * `confidence: number` (match percentage)
  * `onApply: (text: string) => void` (click action to append text)
  * `onCopy: (text: string) => void`
* **Variants**:
  * Standard: Bordered white/dark-slate card.
  * High-Confidence: Glow highlight border.
* **Dependencies**: Action icons (copy, apply)
* **Responsive Behavior**: Horizontal layout on desktop, stacks actions vertically on mobile screens.
* **Theme Behavior**: Gradients transition to dark highlights (`indigo-900/20`) in dark mode.
* **RTL Behavior**: Mirrors action button alignment (Apply button shifts to the left side).
* **Accessibility Requirements**: Card requires `aria-label="AI suggestion"`; button uses keyboard focus indicators.
* **Complexity**: Low

---

## 3. Transfer Modal (`TransferModal`)
* **Purpose**: Provides directory search and buttons to transfer an active case to other agents or queues.
* **Props**:
  * `isOpen: boolean`
  * `onClose: () => void`
  * `onTransfer: (agentId: string, notes: string) => void`
  * `agents: Agent[]` (list of online agents)
* **Variants**: None (standard modal component).
* **Dependencies**: Modal primitive, input fields, agent listings
* **Responsive Behavior**: Scaled down modal box with full width on viewports `<640px`.
* **Theme Behavior**: Dark slate backing with contrasting divider lines in dark mode.
* **RTL Behavior**: Aligns agent names and search text inputs to the right; flips close button placement.
* **Accessibility Requirements**: Implements tab focus trap; keyboard `ESC` key triggers close; elements use labels.
* **Complexity**: Medium

---

## 4. Consult Drawer (`ConsultDrawer`)
* **Purpose**: Drawer panel enabling a side chat or voice consultation with another agent before initiating a transfer.
* **Props**:
  * `isOpen: boolean`
  * `onClose: () => void`
  * `consultTarget: Agent` (agent details)
  * `onExecuteTransfer: () => void`
* **Variants**:
  * Chat Consult: Shows active message composer stream.
  * Voice Consult: Shows call status and dialing indicators.
* **Dependencies**: Drawer wrapper component, message timeline
* **Responsive Behavior**: Slides in full-width on mobile viewports; takes up 320px width on desktop.
* **Theme Behavior**: Purple tinted borders to differentiate from primary conversation thread.
* **RTL Behavior**: Panel slides in from the left instead of the right side in Arabic mode.
* **Accessibility Requirements**: Uses `dialog` role attributes; title explicitly read on drawer open events.
* **Complexity**: High

---

## 5. Participant List (`ParticipantList`)
* **Purpose**: Renders active individuals inside a conference voice session.
* **Props**:
  * `participants: Participant[]`
  * `onMuteParticipant: (id: string) => void`
  * `onRemoveParticipant: (id: string) => void`
* **Variants**: None (standard roster list).
* **Dependencies**: Roster UI blocks and action icons
* **Responsive Behavior**: Adapts fluidly within the parent container space.
* **Theme Behavior**: Renders participant names in clean contrasting typography.
* **RTL Behavior**: Placements of control actions (mute/remove) mirrored.
* **Accessibility Requirements**: Interactive elements require `aria-label` identifying the targeted participant name.
* **Complexity**: Low

---

## 6. SLA Banner (`SLABanner`)
* **Purpose**: Ticking deadline alert pinned to the top of active chats.
* **Props**:
  * `deadline: string` (absolute target date-time)
  * `status: 'within_limit' | 'warning' | 'breached'`
* **Variants**:
  * Normal: Green border, subtle clock icon.
  * Warning: Amber border with slow pulse animation.
  * Breached: Red background, bold white text, fast warning flash.
* **Dependencies**: Clock icon
* **Responsive Behavior**: Absolute stretch matching conversation container width.
* **Theme Behavior**: High-contrast colors matched to dark mode targets.
* **RTL Behavior**: Reverses icons and text positioning.
* **Accessibility Requirements**: Announcements triggered when status transitions from normal to warning.
* **Complexity**: Medium

---

## 7. AUX Selector (`AUXSelector`)
* **Purpose**: Selector for break statuses (Lunch, AUX Break, Training) with built-in duration timer.
* **Props**:
  * `currentStatus: string`
  * `duration: number` (duration in seconds)
  * `onStatusChange: (status: string) => void`
* **Variants**:
  * Normal: Subtle gray drop indicator.
  * Adherence Breached: Flashing amber/red outline indicator.
* **Dependencies**: Selector icons and dropdown controls
* **Responsive Behavior**: Collapses to icon and timer only on narrow viewports.
* **Theme Behavior**: Slate backgrounds with clean borders in dark mode.
* **RTL Behavior**: Dropdown items aligned right; drop arrow positioned left.
* **Accessibility Requirements**: Selector has aria-haspopup and displays status updates in aria-live regions.
* **Complexity**: Medium

---

## 8. Queue Card (`QueueCard`)
* **Purpose**: Displays metrics and status for individual support queues.
* **Props**:
  * `queueName: string`
  * `waitingCalls: number`
  * `averageWaitTime: string`
  * `onSelect: () => void`
  * `isActive: boolean`
* **Variants**:
  * Active: Blue border outline with active indicator dot.
  * Default: Clean slate border.
* **Dependencies**: Layout cards and typography styles
* **Responsive Behavior**: Flex-row on wide viewports, collapses into card layout on mobile.
* **Theme Behavior**: Highlight background states applied to active cards in dark mode.
* **RTL Behavior**: Metrics values text aligned to the left of card headers.
* **Accessibility Requirements**: Entire card behaves as a button with keyboard focus indicator.
* **Complexity**: Low

---

## 9. Customer Context Card (`CustomerContextCard`)
* **Purpose**: Shows essential customer profile information inside the sidebar.
* **Props**:
  * `name: string`
  * `email: string`
  * `loyaltyTier: 'VIP' | 'Gold' | 'Standard'`
  * `avgCSAT: number`
* **Variants**:
  * VIP: Gold background badge and star icon details.
  * Standard: Muted gray text highlights.
* **Dependencies**: SentimentBadge, metadata lists
* **Responsive Behavior**: Width changes to fit side drawer spaces.
* **Theme Behavior**: VIP labels utilize gold text gradients.
* **RTL Behavior**: Text aligned right; labels mirrored.
* **Accessibility Requirements**: Customer loyalty levels announced explicitly by screen readers.
* **Complexity**: Low

---

## 10. Escalation Badge (`EscalationBadge`)
* **Purpose**: Inline indicator signaling a case has been flagged for escalations.
* **Props**:
  * `isEscalated: boolean`
  * `reason?: string`
* **Variants**:
  * Active: Red block with alert symbol.
  * None: Not rendered.
* **Dependencies**: Alert icons
* **Responsive Behavior**: Fixed sizing details.
* **Theme Behavior**: Red visual alert matches light/dark background themes.
* **RTL Behavior**: Placed to the left of the text label.
* **Accessibility Requirements**: Announce "Escalated case" when rendered.
* **Complexity**: Low

---

## 11. Message Composer (`MessageComposer`)
* **Purpose**: Central interface for writing messages, notes, using macros, and adjusting AI parameters.
* **Props**:
  * `value: string`
  * `onChange: (val: string) => void`
  * `onSend: (text: string, type: 'chat' | 'note') => void`
  * `activeTab: 'customer' | 'note'`
  * `onTabChange: (tab: 'customer' | 'note') => void`
* **Variants**:
  * Public Chat: Solid blue action buttons.
  * Internal Note: Deep purple text area highlighting with purple action selectors.
* **Dependencies**: Dropdowns, input elements, suggestion triggers
* **Responsive Behavior**: Adapts size to main panel; text options collapse to mobile menu sheet under `768px`.
* **Theme Behavior**: Inputs change to dark border styles in dark mode.
* **RTL Behavior**: Mirrors text entries and send button positioning.
* **Accessibility Requirements**: Input has accessible name; controls accessible via tab indices.
* **Complexity**: High

---

## 12. Voice Toolbar (`VoiceToolbar`)
* **Purpose**: Docked control bar containing core call action buttons.
* **Props**:
  * `status: CallStatus`
  * `isMuted: boolean`
  * `isHeld: boolean`
  * `isRecording: boolean`
  * `actions: TelephonyActions` (callbacks)
* **Variants**:
  * In-Call: Full option roster visible.
  * Ringing: Only green Accept and red Decline icons rendered.
* **Dependencies**: Action icons (mute, hold, recording, hangup)
* **Responsive Behavior**: Fluid scaling on touch viewports; increases spacing of touch targets to `44px`.
* **Theme Behavior**: Colors match standard active, warning, or danger patterns.
* **RTL Behavior**: Mirrors icon ordering.
* **Accessibility Requirements**: Toggle buttons require `aria-pressed` state properties; icon buttons have tooltips.
* **Complexity**: Medium

---

## 13. Hold Indicator (`HoldIndicator`)
* **Purpose**: Visual banner showing when an active call is placed on hold.
* **Props**:
  * `isHeld: boolean`
  * `duration: number` (hold duration in seconds)
  * `holdMusicName: string`
* **Variants**: None (standard banner).
* **Dependencies**: Audio track indicators
* **Responsive Behavior**: Stretched top banner layout.
* **Theme Behavior**: Highlighted amber warning state indicators.
* **RTL Behavior**: Text and timer alignment mirrored.
* **Accessibility Requirements**: Announces "Call on hold" on activation.
* **Complexity**: Low
